# Data Model Definition (FJH-92)

## 📋 Overview
This task defines the core data architecture for the Flame Japanese Hibachi platform. It encompasses all entity relationships, field definitions, and multi-store data isolation logic.

## 🔗 Detailed Documentation
The comprehensive schema definition is maintained in the roadmap directory:
👉 **[Full Data Model Specifics](../roadmap/DATA_MODEL.md)**

## ✅ Key Components Defined
- **User Identity & Roles**: Auth accounts, staff permissions, and customer profiles.
- **Shop & Location**: Multi-branch support with operating hours and settings.
- **Menu System**: Categories, items, modifiers, and store-specific overrides.
- **Order Flow**: Cart items, payment details, and fulfillment statuses.
- **Analytics**: Pre-computed snapshots for performance monitoring.
